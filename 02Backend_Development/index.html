Basics of Node.js

Node.js is a cross-platform, open-source runtime environment that allows you to run JavaScript code outside of a browser. It is built on Googleâ€™s V8 JavaScript engine and is designed for developing scalable network applications. Node.js is event-driven, non-blocking, and well-suited for handling asynchronous operations efficiently.

Understanding Node.js Runtime

The Node.js runtime refers to the environment where JavaScript code executes. Unlike browsers, Node.js provides additional APIs such as `fs` (file system) and `http` to enable server-side scripting. It leverages the V8 engine for executing JavaScript and uses the libuv library for handling asynchronous I/O operations.

Creating a Simple Server

Creating a simple server in Node.js involves using the built-in `http` module to listen for incoming client requests and respond to them. A basic server can be created with just a few lines of code:


const http = require('http');

const server = http.createServer((req, res) => {
    res.writeHead(200, { 'Content-Type': 'text/plain' });
    res.end('Hello, world!');
});

server.listen(3000, () => {
    console.log('Server running on http://localhost:3000');
});

Understanding Event-Driven Architecture

Node.js operates on an event-driven architecture where an event loop continuously listens for and processes events such as HTTP requests, database queries, or file operations. Instead of waiting for a task to complete, Node.js registers callbacks for events and moves on to the next task. This approach ensures high performance and scalability by minimizing blocking operations.

Example:


const fs = require('fs');

fs.readFile('example.txt', 'utf-8', (err, data) => {
    if (err) throw err;
    console.log(data);
});

console.log('Reading file...');
